### Level 1 - Extend the current model

With the existing one-step LSTM, implement recursive multi-step forecasting:

- Start from the last available test window.
- Predict t+1.
- Insert the prediction back into the rolling window.
- Predict t+2.
- Continue up to t+24, or another forecast horizon.

Evaluate how error grows as the horizon increases:

- MAE(h=1), MAE(h=2), ..., MAE(h=24)

This helps explain:

- how forecast quality degrades as the horizon becomes longer;
- how a one-step model can be used for longer forecast simulations.

### Level 2 - Direct 24-hour model

As the next step:

- Change the final layer so it returns a vector of length 24 instead of a single value.
- Prepare the data so the input is 168 hours of history and the target is the next 24 hours.

Compare:

- one-step LSTM used recursively 24 times;
- multi-output LSTM that forecasts the next 24 hours directly.
